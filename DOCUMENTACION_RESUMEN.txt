================================================================================
                      CÓMO USAR LA DOCUMENTACIÓN
================================================================================

OPCIÓN 1: VISUALIZAR LOCALMENTE (RECOMENDADO)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Instalar MkDocs:
   $ pip install mkdocs mkdocs-material pymdown-extensions

2. Navegar a la carpeta del proyecto:
   $ cd /vercel/share/v0-project

3. Servir la documentación:
   $ mkdocs serve

4. Abrir navegador:
   http://localhost:8000

5. Listo! Los cambios en .md se reflejan automáticamente

OPCIÓN 2: COMPILAR PARA PRODUCCIÓN
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

$ mkdocs build
→ Genera: site/ (HTML estático)

OPCIÓN 3: PUBLICAR EN INTERNET
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

GitHub Pages:
$ mkdocs gh-deploy

ReadTheDocs (recomendado):
1. Conectar repo a ReadTheDocs
2. Automáticamente detecta mkdocs.yml
3. Publica en cada push

Vercel:
1. Crear vercel.json con buildCommand: "mkdocs build"
2. Conectar repo
3. Deploy automático
